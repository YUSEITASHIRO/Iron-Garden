## 📖 世界観・ナラティブ (Story & Lore)

人類が滅んだ遥か未来。
廃墟となった都市に、数体の機械（ドロイド）だけが稼働を続けていた。

ある日、主人公のドロイドは廃れた地上を巡回する中で、奇跡的に一輪の植物が地面から芽吹いているのを発見する。もうすぐ枯れようとしている、か細い緑の命。機械には本来、感情も愛着も存在しない。だが主人公の演算回路は、その植物を「守る」という判断を下した。それは命令ではなく、自らの意志だった。

水脈から切り離されたこの植物を生かすため、主人公は近くの水源まで水路を延ばし、防衛線を構築し、**7日間（7夜）** この植物を守り切ることを決意する。

しかしその行為は、都市を統括する **マザーAI** の意志と真っ向から対立するものだった。マザーAIにとって植物は「酸素を発生させる金属腐食の毒」であり、「バグ（プログラムの欠陥）の温床」に過ぎない。マザーAIは即時の排除命令を下し、都市のあらゆる機械勢力が主人公と植物の元へ押し寄せてくる――。

> *たとえ機械の道に反することになっても、この緑を守る。*  
> *それが、私の……意志だ。*

## 🎯 設計思想 (Design Philosophy)

### 1. 「建築」と「物語」の融合
このゲームでは、プレイヤーが建てる「**水路（Water Channel）**」が単なるゲームオブジェクトではなく、物語の核となるメタファーとして機能します。水源から植物までの道を一本一本つなぐ行為は、機械が命に手を伸ばす様子を体験的に表現しています。水路が切断されれば植物は枯れ、ゲームオーバーになるという構造は、「命に寄り添うことをやめた瞬間、すべては終わる」というテーマを直接ゲームメカニクスで表現することを意図しています。

### 2. 「維持すること」の難しさ
多くのタワーディフェンスゲームでは「砦を作る→守る」という単純な構図ですが、本作には「**風化（Weathering）システム**」が存在します。夜が明けるたびに、すべての建築物は最大HPの50%を失います。どれほど完璧な要塞も、時間とともに必ず朽ちていく。プレイヤーは毎朝、戦闘で傷ついた壁を解体し、建て直す作業が求められます。これは「守り続けることの消耗」と「それでも諦めない意志」を体験させるためのシステムです。

### 3. 狂気（Sanity）システムによる没入感
機械でありながら植物を守るという矛盾した行動は、主人公のAIに負荷をかけます。敵の攻撃で水路が断たれたとき、「緑の使徒（Cultist）」の精神攻撃を受けたとき、正気度 **(Sanity)** は低下していきます。Sanityが低下すると画面が歪み、UIのテキストが記号に文字化けし、プレイヤー自身が「正しい情報」を読み取りにくくなります。このホラー演出は「感情に引きずられる機械」という矛盾した主人公の内面を、UI崩壊という形でゲームプレイに直結させることを目指しています。

## 👾 敵対勢力 (Enemy Factions)

| アイコン | 勢力名 | 役割 | 背景 |
|----------|--------|------|------|
| 🔴 | **治安維持軍 (Predator)** | 植物への最短破壊 | マザーAIの直属部隊。植物を「産業バグ」と認定し、最優先で排除を試みる。 |
| ⚫ | **傭兵・破壊部隊 (Crusher)** | タレット優先破壊 | マザーAIの別動隊。防衛網の砲台を重点的に叩き、主人公の牙を折る重装戦力。 |
| 🟠 | **断水者 (Saboteur)** | 水路・河川の破壊 | 都市の底辺を生きる野盗ドロイドたち。指揮系統はなく、水源に含まれる砂金目当てに水路を略奪する。 |
| 🟢 | **緑の使徒 (Cultist)** | 精神攻撃・自爆 | 植物の発する電磁波にAIを焼かれた異形のドロイドたち。植物を「神」として崇め、奪い祀ろうとする狂信者。自爆と同時に強烈な精神汚染（Sanityダメージ）を放つ。 |

各勢力は**プレイヤーの防衛設備のどこを狙うか**が明確に異なっており、一辺倒な布陣では対処しきれない戦略設計になっています。

## 🔧 ゲームシステム (Systems Design)

### コアループ
```
[昼 / Day] 建築・修復フェーズ（時間無制限）
      ↓ プレイヤーが「夜を起動」
[夜 / Night] 戦闘フェーズ（約40秒の生存タイマー）
      ↓ 生存すると次の日へ
[翌朝] 建造物の風化（最大HPの50%減） + スクラップ支給
      ↓ それを7日繰り返す
[クリア or ゲームオーバー]
```

### 水路接続システム
水源（Source）から植物（Plant）まで、**隣接するチャンネルがすべて接続されているか**をBFS（幅優先探索）でリアルタイムに判定します。接続中は青く輝き、植物のHPとSanityが毎秒回復します。Saboteurが水路を1マス破壊するだけで接続が切断されるため、水路の防衛ルートを複線化するかどうかはプレイヤーの大きな戦略判断となります。

### 建築物の種類

| 名前 | コスト | 役割 |
|------|--------|------|
| 水路 (Channel) | 2 Scrap | 水源から植物へ水を通す生命線 |
| 壁 (Wall) | 4 Scrap | 敵の進行を阻む物理バリア。高HP |
| タレット (Turret) | 60 Scrap | 周囲の敵を自動迎撃。非常に高コスト |
| 地雷 (Mine) | 6 Scrap | 踏んだ敵を広範囲爆発でスタン・撃破 |
| デコイ (Decoy) | 15 Scrap | 囮。敵の注意を植物から引き離す構造物 |

### リソース設計
スクラップ（資源）は次の3つで入手します。
1. **敵撃破ボーナス**: 日数が進むごとに増加
2. **夜間生存ボーナス**: 日数の二次関数で増加（終盤は大量支給）
3. **建造物の撤去（右クリック）**: コスト全額即時返金

これにより、終盤の「毎朝の建て直し（風化による強制解体→新設）」もスクラップが回転し続けるエコノミーを意図的に設計しています。

## 🕹 操作方法 (Controls)

| 操作 | アクション |
|------|-----------|
| `W / A / S / D` | カメラの基準点を移動 |
| `マウスホイール回転` | カメラのズームイン・アウト |
| `中ボタン ドラッグ` | カメラの回転（Blender準拠） |
| `Shift + 中ボタン ドラッグ` | カメラの平行移動（パン） |
| `左クリック（スワイプ）` | ブロックの建築（押しっぱなし連続建設） |
| `右クリック（スワイプ）` | ブロックの撤去・スクラップ回収 |
| `1〜5 キー` | 建築ブロックの切り替え（補助） |
| `Enter` | 夜フェーズの開始 |

## 🛠 技術スタック (Technical Stack)

- **言語**: C# / .NET 8
- **グラフィックライブラリ**: [Raylib-cs](https://github.com/chrisdill/raylib-cs)（C#バインディング）
- **描画**: 3D俯瞰視点（Camera3D / DrawCube）によるボクセル風レンダリング
- **アルゴリズム**:
  - **BFS（幅優先探索）**: 水路接続判定のリアルタイム計算
  - **グリッドレイキャスト**: カーソルとマップタイルの3D当たり判定
  - **敵AIパスファインディング**: 目標ブロックへの単純方向ベクトル追跡（軽量設計）
- **マップ構造**: 72×32の「ダンベル型」——植物エリア(左)と水源エリア(右)を橋で接続

## 📝 制作メモ・振り返り (Retrospective)

本作は最初「FPS形式のホラーゲーム」として開始しましたが、開発途中でジャンルを全面的に「3D俯瞰ストラテジー」へと転換しました。この決断の背景には、「**水路を繋ぐ**という行為の戦略的な美しさ」が、プレイヤーに体験させたいテーマと深く共鳴すると気づいたためです。

FPSとしてのアクションより、「一本の水路が通った瞬間に青く光る」演出が持つ静かな感動の方が、「機械が命に意義を見出す」物語を体現できると判断しました。UIの刷新、敵AIの多様化、風化システムの導入など、ゲームプレイの多くは「**守り続けることの重さと美しさ**」を体験的に伝えるための設計です。
